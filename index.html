an insight into data structure

What is "data structures"?!

As pictures illustrate, We see Data structure all around us. we actually use its methods in our everyday life, at our jobs and houses to organize and store items. Data structures enable us not only to store and organize our data, but also we have the possiblity to access and modify them efficiently. Arrays, Linked Lists, trees and Queues are covered here. A short description of each data structure is given at its relative section. A more in-depth explanation and code implementation approach is available at my GitHub page.

1. Basic Array

Arrays are mainly used because of their structure to store elements. First element has the base address. You can access each element of array using indexing. Different kind of arrays can be implemented from one dimensional to multidimensional arrays. Arrays can also be static or dynamic. Arrays are used in implementation of lists, queues, heaps, hash tables for example.

2. Queue

Queue is much like what we understand as lining up. It is a sequence of elements. New element can be added to back of the sequence (tail) and element can be removed from front of the sequence (head). The process of removing from the sequence follows FIFO ( first in first out).

3. Trees

Trees are structured as linked nodes. The very first node of a tree is root node. Important terms are used in working with trees, but the most important ones are child nodes, parent nodes, sibling nodes. A subtree can also be defined from a tree. Subtree is also a tree from the main tree, which consists of a node and all of desendants of that node.

4. Linked list

In comparison to array, elements of a linked lists are not structured based on their physical placement. The elements points to each other. Therefore it represents a sequence. Linked lists are for example used to implement queues, stacks, binary trees, hash table. Linked lists are categorized as :

  • Single linked list
  • Doubly linked list
  • Multiply linked list
  • Circular linked list
